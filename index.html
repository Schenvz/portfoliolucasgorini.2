React · Frontend
Cómo pasé de no entender un componente a construir apps completas en React en 3 meses
Lucas Gorini 5 min de lectura 2026

Cuando vi React por primera vez pensé que era magia negra. Props, estado, hooks, efectos — todo junto al mismo tiempo, sin entender por qué nada funcionaba como yo esperaba. Los primeros días fueron frustrantes. Copiaba código de tutoriales, lo pegaba, funcionaba, y yo no entendía absolutamente nada de lo que había hecho.

El momento que todo cambió

Lo que cambió todo fue una decisión simple: dejar de copiar tutoriales y empezar a romper cosas. Eliminaba líneas de código al azar para ver qué se rompía. Cambiaba valores para ver qué pasaba. Me equivocaba, leía el error en la consola, buscaba por qué, y lo arreglaba. Ese ciclo repetido cientos de veces fue lo que me hizo realmente entender React.

De cero a app fullstack en 3 meses

En 3 meses pasé de no entender qué era un componente a tener una app fullstack desplegada en Vercel con autenticación con JWT, rutas dinámicas, manejo de estado global con Context API, y conexión a base de datos real con PostgreSQL.

¿La lección más importante? Construir cosas feas que funcionen antes de construir cosas lindas que no sirvan para nada. El perfeccionismo al principio mata el aprendizaje. Mejor tener algo feo funcionando que algo hermoso que nunca termina.

Lo que recomendaría a alguien empezando

No hagas 10 cursos de React. Hace un proyecto real aunque sea horrible. Rompelo. Arreglalo. Volvé a romperlo. En ese proceso está todo el aprendizaje real que ningún tutorial te va a dar.

Node.js · Backend · APIs
El día que construí mi primera API REST desde cero y entendí cómo funciona internet
Lucas Gorini 7 min de lectura 2026

Durante mucho tiempo usé APIs de terceros como si fueran cajas negras. Stripe para pagos, Firebase para auth, alguna API de clima — las consumía sin entender absolutamente nada de lo que pasaba por adentro. Solo sabía que hacías un fetch y algo mágico volvía.

Construyendo mi primera API real

El día que decidí construir la mía propia con Node.js y Express fue un antes y un después. De repente entendí qué es una request HTTP, qué es una response, qué son los headers, por qué existe el status code 200, cuándo usar 201, y por qué 401 no es lo mismo que 403 (uno significa que no estás autenticado, el otro que estás autenticado pero no tenés permiso).

JWT, middlewares y PostgreSQL

Agregué autenticación con JWT y entendí por qué una contraseña nunca debería guardarse en texto plano — implementé bcrypt para hashearlas. Agregué middlewares para validar tokens en cada ruta protegida. Conecté todo a PostgreSQL y escribí mis primeras queries SQL reales.

Lo que cambió para siempre

Ahora cuando uso una API ajena sé exactamente lo que está pasando del otro lado. Sé cómo están validando mi token, cómo están guardando mis datos, qué error me están mandando y por qué. Ese conocimiento vale oro — te convierte en un desarrollador que realmente entiende el sistema completo, no solo la parte que ve en pantalla.
